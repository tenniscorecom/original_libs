# コードリーディングガイド（レビュー用）

このライブラリ（comken）をレビュー・読解する人向けの地図。
「どこから読めばいいか」「どのファイルも同じ形になっている」ことを先に押さえると速く読める。

- 何のライブラリか: 業務自動化（RPA・日次バッチ）の共通部品集。
  CSV/Excel の読み書き、Salesforce API、ブラウザ操作、認証情報の暗号化保存を、
  非エンジニアでも使い方を真似できる形でまとめたもの
- 規約: [CONVENTIONS.md](../CONVENTIONS.md)（命名・型ヒント・例外・ロギング等はすべてここに準拠）
- 使い方の説明: [README.md](../README.md)（利用者向け）。本書はコードを読む人向け

## 全体地図

| パッケージ | 役割 | 主役クラス・関数 | 外部依存 |
|---|---|---|---|
| `exceptions.py` | 例外の全体系（すべて `OriginalLibsError` 基底） | — | なし |
| `runtime.py` | dry-run / debug のグローバルスイッチ | `set_dry_run` ほか | なし |
| `logger.py` | ログ設定（コンソール + 日付付きログファイル） | `setup_logger` | なし |
| `config.py` | config.ini 読み込み（`config.SECTION.KEY` アクセス） | `Config` | なし |
| `utils/` | 汎用部品（ファイル探索・差分比較・リトライ等） | `FileFinder`, `diff_rows`, `retry` | なし |
| `csv/` | CSV 読み書き（文字コード自動判定） | `CsvReader`, `CsvWriter` | なし |
| `excel/` | Excel 読み書き（openpyxl。Excel 本体不要） | `ExcelFile`, `Sheet` | openpyxl |
| `windows/` | Excel COM（数式・マクロ・パスワード保存）、ウィンドウ・レジストリ | `ExcelComHandler` | pywin32 |
| `browser/` | ブラウザ自動操作（Page Object Model の土台） | `EdgeDriver`, `BasePage` | selenium |
| `salesforce_requests/` | Salesforce API（requests 版） | `SalesforceApiClient` | requests |
| `salesforce_std/` | 同上の標準ライブラリのみ版（依存を入れられない環境用） | `SalesforceApiClient` | なし |
| `salesforce/` | 旧 import パスの互換シム（新規コードでは使わない） | — | — |
| `credentials/` | 機密情報の暗号化保存（Windows DPAPI）+ 登録 GUI | `Credentials` | pywin32 |
| `teams/` | Teams 通知（Webhook） | `TeamsNotifier` | なし（urllib） |
| `pdf/` | PDF 結合・分割・テキスト抽出 | `merge_pdfs` ほか | pypdf |

### 依存の向き

```
exceptions.py / runtime.py / utils/   ← 土台。どこからも参照される
        ▲
各機能パッケージ（csv, excel, windows, browser, salesforce_*, ...）
```

- 機能パッケージ同士は原則独立。例外は1つだけ:
  `excel/`（openpyxl）が数式計算・マクロ実行時に `windows/`（COM）へ
  **遅延 import でフォールバック**する（pywin32 がない環境でも excel/ 単体は動く）
- `salesforce_std/` は `salesforce_requests/` と同じクラス名・同じメソッド。
  requests を入れられない環境向けの複製で、import 行の変更だけで切り替えられる

## 読む順番（おすすめ）

1. **[CONVENTIONS.md](../CONVENTIONS.md)** — 全ファイル共通のルール。これを先に読むと以降が速い
2. **`exceptions.py`** — 例外の全体系が1ファイルに集まっている。エラーメッセージの方針
   （非エンジニアが読んで対処できる文面）もここで分かる
3. **`runtime.py` → `logger.py` → `config.py`** — 土台となる小さいモジュール3つ
4. **`csv/handler.py` → `excel/handler.py`** — 一番よく使われる部分。
   ここで「docstring の書き方」「定数クラス」「with 文」のパターンをつかむ
5. **`windows/handler.py`** — COM 版 Excel。openpyxl 版との役割分担が分かる
6. **`credentials/store.py`** — 機密の扱い（DPAPI）。セキュリティ観点のレビューはここが本丸
7. 残り（`browser/`, `salesforce_*`, `teams/`, `pdf/`, `utils/`）は必要に応じて

## どのファイルも同じ形

全モジュールが同じ並び順で書かれている（詳細: CONVENTIONS.md「モジュール内の並び順」）:

```
1. モジュール docstring   ← そのファイルの役割と使い方（コピペで動く例）
2. import
3. logger・型変数
4. 定数・定数クラス
5. 主役の公開クラス       ← モジュール名が指すもの
6. その他の公開クラス・公開関数
7. 内部ヘルパー（_ 付き）  ← 必ず最後
```

つまり**ファイル先頭の docstring と主役クラスの docstring だけ読めば、そのモジュールの
全体像と使い方が分かる**。細部（内部ヘルパー）は必要なときだけ下へ読み進めればよい。

## 設計上の約束事（レビュー観点の要約）

| 観点 | 方針 | 確認場所 |
|---|---|---|
| 例外 | 素の Exception を投げない。全例外が `OriginalLibsError` 基底。メッセージは「何が・どこで・どうすればよいか」 | `exceptions.py` |
| リソース解放 | ファイル・COM・WebDriver はすべて `with` 対応（`__enter__`/`__exit__`） | 各 handler |
| 機密情報 | config.ini に書かない。DPAPI で暗号化しユーザープロファイル内に保存 | `credentials/store.py` |
| ログ | `print` 禁止。`logging` のみ。機密の値はログに出さない | 全ファイル |
| dry-run | `set_dry_run(True)` で書き込み系操作をログ出力だけにできる（本番前の確認用） | `runtime.py` |
| 型ヒント | 全関数に引数・戻り値の型。`Any` 不使用 | 全ファイル |
| 静的チェック | Ruff（line-length 100, py310, E/F/W/I）全ファイルパス | `pyproject.toml` |
| 破壊的変更 | 旧 API は互換シム + `FutureWarning` で残す（例: `comken.salesforce`） | `deprecation.py` |

## テスト

```
python -m pytest tests -q    # 292件 / 約5秒
```

- 単体テストは pytest。一時ファイルは `tmp_path` フィクスチャで完結し、外部環境不要
- **対象外**: Excel COM の実動作（Excel 本体が必要）、ブラウザ実機、Salesforce 実接続。
  これらは利用プロジェクト側の初回実行時に動作確認する運用

## 想定問答

**Q. なぜ Salesforce のフォルダが3つあるのか？**
`salesforce_requests`（requests 版・推奨）と `salesforce_std`（標準ライブラリのみ・
依存を入れられない環境用）は同一 API の2実装。`salesforce` は旧コードを壊さないための
互換シムで、警告を出しつつ新モジュールへ誘導する。

**Q. Excel の処理が openpyxl と COM の二本立てなのはなぜ？**
通常の読み書きは openpyxl（Excel 本体不要・高速・サーバーでも動く）。
数式の再計算・マクロ実行・パスワード付き保存だけは Excel 本体の機能が必要なので
COM（`ExcelComHandler`）を使う。使い分けは各クラスの docstring 冒頭に明記してある。

**Q. パスワード類はどこに保存されるのか？**
`python -m comken.credentials` で登録し、Windows DPAPI で暗号化して
`%USERPROFILE%\.comken\credentials.dat` に保存される。同じ「ユーザー × PC」でしか
復号できず、ファイルをコピーされても中身は読めない。コード・config.ini・git には残らない。

**Q. 認証情報を複数PCへ配る場合は？**
[認証情報の公開鍵配布.md](認証情報の公開鍵配布.md) と
[Salesforce認証パターン.md](Salesforce認証パターン.md) に選択肢と設計を整理してある（検討中）。
