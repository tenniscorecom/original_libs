# 機能カタログ — 用途別 API 一覧

「やりたいこと」から、**用意されている関数**と**書き方**を引くためのドキュメント。
一覧性を優先している。詳しい説明と動く例は [README.md](../README.md) の各節と `examples/` を参照。

## CSV を読む — CsvReader

`CsvReader(path, encoding=Encoding.AUTO, headers=None)`
文字コードは自動判定（UTF-8 → CP932 の順）。ヘッダーがない CSV は `headers=[...]` で列名を付ける。

| メソッド | 何をするか |
|---|---|
| `rows(columns=None)` | 全行を辞書のリストで返す（`columns` で列を絞れる） |
| `find(key_col, value)` | キーに一致する最初の1行を返す（なければ None） |
| `filter(key_col, value)` | キーに一致する行すべてを返す |
| `column(col_name)` | 1列だけをリストで返す |
| `index(key_col)` | `{キーの値: 行}` の辞書を作る（転記の lookup 用） |

```python
from comken.csv import CsvReader

rows = CsvReader(r"C:\作業\data.csv").rows()          # [{"注文番号": "A001", ...}, ...]
lookup = CsvReader(r"C:\作業\data.csv").index("注文番号")  # 転記用の辞書
```

## CSV を書く — CsvWriter

`CsvWriter(path, fieldnames, encoding=Encoding.UTF8_SIG)`
デフォルトは BOM 付き UTF-8（Excel でダブルクリックしてもそのまま開ける）。

| メソッド | 何をするか |
|---|---|
| `write_rows(rows)` | ヘッダー + 全行を新規書き込み |
| `append_row(row)` / `append_rows(rows)` | 既存ファイルに追記 |

## Excel を読み書きする — ExcelFile / Sheet

`ExcelFile(path)` で開き、`with` 文で使う。Excel 本体は不要（openpyxl）。
**書き込み後は `save()` を忘れない**こと。

| ExcelFile のメソッド | 何をするか |
|---|---|
| `ExcelFile.create(path)` | 新規ブックを作る（save() 時にファイルが作られる） |
| `sheet(name)` | シートのラッパー（下の Sheet）を返す |
| `read_rows(sheet, min_row=2)` | 行データをタプルのリストで返す |
| `read_rows_as_dicts(sheet)` | ヘッダーをキーにした辞書のリストで返す |
| `iter_rows(sheet)` | 1行ずつ返すジェネレータ（数万行のファイル向け） |
| `read_computed_rows(sheet)` | 数式の**計算結果**を読む（必要時のみ COM に自動切替） |
| `write_cell(sheet, row, col, value)` | セルに書き込む |
| `transfer_by_key(sheet, key_col, lookup, column_mapping)` | キー突合で転記（XLOOKUP 的転記） |
| `set_fill` / `set_bold` / `set_number_format` / `set_column_width` | 書式設定 |
| `run_macro(name)` | VBA マクロ実行（COM に自動切替） |
| `save(path=None)` | 保存（省略で上書き、パス指定で別名） |

| Sheet のメソッド | 何をするか |
|---|---|
| `s["A1"]` / `s["A1"] = 値` | セル参照で読み書き |
| `write_row(row, values)` / `write_rows(start_row, rows)` | 行単位で書く |
| `append_row(values)` | 最終行の下に追記 |
| `write_table(rows)` | 辞書のリストを「ヘッダー + データ」でまとめて書く |
| `auto_width()` | 列幅を内容に合わせる（全角対応） |
| `freeze_header()` | ヘッダー行を固定 |
| `last_row` / `is_empty` | 最終行番号 / 空シート判定 |
| `.ws` | 生の openpyxl Worksheet（ここにない操作用） |

```python
from comken.csv import CsvReader
from comken.excel import ExcelFile

rows = CsvReader(r"C:\作業\data.csv").rows()

with ExcelFile.create(r"C:\作業\report.xlsx") as f:   # 新規作成
    s = f.sheet("Sheet1")
    s.write_table(rows)      # CSV の中身をそのまま表に
    s.auto_width()
    s.freeze_header()
    f.save()

with ExcelFile(r"C:\作業\台帳.xlsx") as f:             # 既存ファイルに転記
    lookup = CsvReader(r"C:\作業\data.csv").index("注文番号")
    f.transfer_by_key("T_data", key_col="Q", lookup=lookup,
                      column_mapping={"A": "顧客名", "B": "金額"})
    f.save()
```

## Excel の数式・マクロ・パスワード保存 — ExcelComHandler

Excel 本体を起動して操作する（pywin32）。**openpyxl でできることはExcelFile を使い、
こちらは数式の再計算・マクロ・パスワード保存が必要なときだけ**使う。

`ExcelComHandler(path, password="")` — パスワード保護されたファイルも開ける。

| メソッド | 何をするか |
|---|---|
| `read_cell` / `write_cell` / `read_rows` / `read_rows_as_dicts` | ExcelFile と同じ感覚で読み書き（数式は計算結果） |
| `used_last_row(sheet)` | データのある最終行 |
| `count_a(sheet, row)` | 行の空でないセル数（0 なら空行） |
| `transfer_by_key(...)` | キー突合転記（COM 版） |
| `run_macro("Module1.名前")` | VBA マクロ実行 |
| `save()` | **上書き保存**（close() は保存しないので、変更を残すなら必須） |
| `save_as(path, read_pw="", write_pw="", file_format=None)` | 別名保存。読取/書込パスワード設定可。形式は元ファイルと同じ（変換時のみ `FileFormat.CSV` 等を指定） |

```python
from comken.windows import ExcelComHandler

with ExcelComHandler(r"C:\作業\data.xlsm") as h:
    h.run_macro("Module1.UpdateData")
    h.save()                                          # 上書き保存
    h.save_as(r"C:\作業\提出用.xlsm", read_pw="1234")  # パスワード付きで別名保存
```

## ファイルを探す・動かす — comken.utils

| 関数・クラス | 何をするか |
|---|---|
| `FileFinder(folder).today(pattern="*.xlsx")` | ファイル名に今日の日付を含むファイルを返す |
| `FileFinder(folder).latest(pattern, by=SortBy.NAME)` | 最新ファイルを返す（名前順 / `SortBy.UPDATED` で更新日時順） |
| `FileNameBuilder("売上").prefix()` | `20260713_売上.xlsx` のような日付付きファイル名を作る |
| `move_file(src, dst)` / `copy_file(src, dst)` | 移動・コピー（dst がフォルダなら中へ、親フォルダ自動作成、上書き） |
| `local_copy(nas_path)` | NAS のファイルをローカルに一時コピーして使う（with 文で自動削除） |
| `Paths.downloads()` / `Paths.desktop()` | 実際の Downloads / Desktop の場所（OneDrive 移動にも追従） |
| `zip_folder(folder)` / `zip_files(files, dst)` / `unzip(src)` | ZIP 圧縮・展開 |

```python
from comken.utils import FileFinder, FileNameBuilder, move_file

src = FileFinder(r"\\nas\share").today(pattern="売上_*.xlsx")  # なければ FileNotFoundError
out = FileNameBuilder("集計結果").prefix()                      # "20260713_集計結果.xlsx"
move_file(src, rf"C:\作業\done\{out}")
```

## データを比較・変換する — comken.utils

| 関数 | 何をするか |
|---|---|
| `diff_rows(before, after, key)` | 2つのデータをキーで突合し、追加/削除/変更を返す（CSV と Excel をまたいでも "1000" と 1000 は同一視） |
| `diff_row(before, after)` | 1行同士の差分（変わった列だけ返る） |
| `col_to_num("Q")` | 列レター → 列番号（Q → 17） |
| `normalize(text)` | 全角半角・空白のゆらぎを揃える |
| `strip_spaces` / `remove_spaces` | 前後の空白除去 / 全空白除去 |

## リトライ・待機・時間計測 — comken.utils

| 関数 | 何をするか |
|---|---|
| `@retry(times=3, wait=1.0)` | 失敗したら待って再実行するデコレータ（ネットワーク・ダウンロード系に） |

## Salesforce を操作する — SalesforceApiClient

`SalesforceApiClient(username, password, security_token, domain="login")`
認証情報は `Credentials`（下記）から渡す。CRUD・SOQL・レポート・Bulk まで1クラスで揃っている。

| メソッド | 何をするか |
|---|---|
| `query(soql)` | SOQL 実行 → 辞書のリスト |
| `get(object, id)` | レコード1件取得 |
| `insert(object, data)` / `update(object, id, data)` / `delete(object, id)` | 1件ずつの CRUD |
| `upsert(object, external_id_field, data)` | 外部キーで挿入 or 更新 |
| `run_report(report_id)` / `run_report_async(...)` | Salesforce レポートの結果を取得 |
| `bulk_insert / bulk_update / bulk_upsert / bulk_delete(object, records)` | 大量データの一括操作（Bulk API 2.0） |
| `bulk_query(soql)` | 大量データの一括取得 |

```python
from comken.credentials import Credentials
from comken.salesforce_requests import SalesforceApiClient

cred = Credentials("salesforce")   # 事前に python -m comken.credentials で登録
sf = SalesforceApiClient(cred.username, cred.password, cred.token)

rows = sf.query("SELECT Id, Name FROM Account WHERE 業種__c = '製造'")
sf.update("Account", rows[0]["Id"], {"電話__c": "052-xxx-xxxx"})
```

- requests を入れられない環境では import を `comken.salesforce_std` に変えるだけ（同じ API）

## ブラウザを自動操作する — comken.browser

必ず Page Object Model で書く（画面ごとにクラスを作る）。selenium を直接 import しない。
雛形は `examples/sample_login/` にある。

**操作メソッドは「操作 × セレクター」の総当たりで用意されている:**

| 操作 \ 指定方法 | `Locator`（推奨） | id | css | xpath |
|---|---|---|---|---|
| クリック | `click(LOC)` | `click_id` | `click_css` | `click_xpath` |
| 文字入力 | `input(LOC, text)` | `input_id` | `input_css` | `input_xpath` |
| テキスト取得 | `text(LOC)` / `texts(LOC)` | `text_id` | `text_css` | `text_xpath` |
| プルダウン選択 | `select_text(LOC, 表示名)` ほか | `select_text_id` … | 〃 | 〃 |
| 表示を待つ | `wait_visible(LOC)` / `wait_invisible(LOC)` | `wait_visible_id` | 〃 | 〃 |
| 存在確認・件数 | `has(LOC)` / `count(LOC)` | `has_id` / `count_id` | 〃 | 〃 |

そのほか: `alert_accept/dismiss/text`（確認ダイアログ）、`scroll_to` / `scroll_bottom`、
`save_screenshot()`（エラー時の証跡）、`js(script)`（最終手段）。

| 部品 | 役割 |
|---|---|
| `EdgeDriver` | with 文で確実に終了する WebDriver。`d.download_dir.wait()` でダウンロード完了待ち |
| `BrowserOptions` | HEADLESS・DOWNLOAD_DIR・WAIT_SECONDS 等の設定（src/browser_options.py に置く） |
| `Locator.id("username")` 等 | セレクターをクラス上部の定数にまとめるための入れ物 |
| `SitePage` | `BASE_URL` + `go(path)` を持つ画面クラスの親 |

```python
from comken.browser import Locator
from .app_page import AppPage

class LoginPage(AppPage):
    PATH = "/login"
    USERNAME = Locator.id("username")     # セレクターはクラス上部にまとめる
    LOGIN_BTN = Locator.css(".radius")

    def login(self, username: str, password: str) -> "DashboardPage":
        from .dashboard_page import DashboardPage
        self.input(self.USERNAME, username)
        self.click(self.LOGIN_BTN)
        return DashboardPage(self._driver)   # 遷移先の画面クラスを返す
```

## 設定・ログ・dry-run — 共通の土台

```python
from comken import Config, setup_logger, set_dry_run

config = Config()                       # config.ini を読む（src/config.py に1回だけ書く）
logger = setup_logger()                 # コンソール + logs/日付.log に出力

config.FILES.INPUT_FOLDER               # → Path（型は自動変換: bool/int/Path/list）

set_dry_run(True)                       # 書き込み系が「何をするか」のログだけになる
```

| 部品 | 何をするか |
|---|---|
| `Config` | config.ini を `config.SECTION.KEY` で読む。エディタ補完スタブも自動生成 |
| `setup_logger()` | プロジェクト標準のログ設定 |
| `set_dry_run(True)` | 保存・移動・送信を実行せずログだけ出す（本番前の確認） |
| `set_debug(True)` | デバッグログを有効化 |

## エラーが出たら

すべての例外は `OriginalLibsError` 基底で、メッセージに対処法が書いてある。
まとめて受けるなら:

```python
from comken.exceptions import OriginalLibsError

try:
    main()
except OriginalLibsError as e:
    logger.error("処理を中断しました: %s", e)
```

エラー名ごとの対処は [ERRORS.md](../ERRORS.md) を参照。
